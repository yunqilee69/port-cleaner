# Port Cleaner 顶部刷新与 Windows 启动修复设计

## 目标

- 刷新进度只在顶部刷新按钮中展示，不在内容区重复显示刷新横幅。
- 顶部仅保留 `Port Cleaner` 品牌名称，移除“本机端口管理工具”和“在线 · 本机”。
- Windows 发布版启动时不创建额外的 CMD 窗口，关闭终端也不再连带关闭应用。
- 增大桌面窗口默认宽度，并明确后续调整位置。

## 方案

前端继续复用现有 `isRefreshing` 状态：刷新按钮在请求期间保持禁用、图标旋转并显示“扫描中…”。删除 `App` 内容区中只服务于刷新的横幅，但保留结束进程成功提示。

`ConsoleHeader` 删除副标题和本机在线状态节点，同时清理不再使用的样式与响应式规则。Tauri 窗口宽度在 `src-tauri/tauri.conf.json` 的 `app.windows[0].width` 中调整，并允许在保持至少 `1000` 的前提下继续增大。

Windows 入口在非调试构建时使用 `windows` 子系统；调试构建仍保留控制台，便于开发期间查看日志。

## 验证

- React 回归测试验证刷新进度只出现在按钮、顶部无冗余文案。
- Node 配置测试验证 Windows 子系统属性存在且仅在非调试构建生效。
- 运行前端测试、发布元数据测试、TypeScript/Vite 构建和 Rust 测试。
